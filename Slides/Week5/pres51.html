Introduction
One area of computational text analysis is known as Natural
Language Processing (NLP).
NLP uses machine learning classiﬁers trained on “naturally-produced”
linguistic data. The goal is to train computers to recognize, and, in a
sense, understand human language based on its semantic features
and syntactical rules.
Some use the labels “computational text analysis” and “NLP”
interchangeably; others see NLP as a subset of the former that
integrates information about grammatical rules.
The sentiment polarity algorithm implemented in the sentimentr
package (Rinker 2018) is an NLP algorithm.

What is NER?
Named-entity recognition (NER) is an NLP tool.
NER classiﬁers input text documents and return the “entities” found
in those texts: e.g., names, organizations, locations, and dates, to
name a few.
NER is useful for information extraction problems and also for helping
construct dictionaries because it can identify the people, places
things, etc., that are referenced the most frequently in a corpus.

How does it work?
NER algorithms diﬀer from one to the next. The entity package we’ll
use today is an R “wrapper” for the Java-based OpenNLP library
maintained by The Apache Software Foundation (2017).
The OpenNLP NER classiﬁer uses what’s called “pre-trained classiﬁers.”
All this means is that each classiﬁer—e.g., the “Person” classiﬁer, the
“Organization” classiﬁer, and so on—has been optimized to ﬁnd entities in
untrained data and tag them as the appropriate type (person vs.
organization) based on linguistic features that it “learned” from a large
collection of pre-classiﬁed sentences.
Instructor: Marshall A. Taylor SOC/MDSC 43919 February 10, 2019 7 / 24
How does it work?
So the training data are those sentences that have already been tagged by
a human (or set of humans). A classiﬁer is built from these training data,
where it learns to classify based on various linguistic features such as
capitalization, context word associations, grammatical patterns, and so on.
This classiﬁer is then applied to your data.
The OpenNLP training data come from a number of sources, including, for
the English classiﬁers, texts from the Wall Street Journal.

How does it work?
Using the patterns found in the training data, the NER algorithm then
outputs the particular “tag sequence” for our document that gives us the
highest conditional probability of correctly predicting the classes to which
each identiﬁed entity in the document belongs (Bender et al. 2003:1;
Berger et al. 1996).
This is known as the “maximum entropy model” of named-entity
recognition.
